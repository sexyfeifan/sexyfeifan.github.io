# 🦞 一“爪”搞定多平台，OpenClaw 到底有多香？

最近在小红书上刷到一个超酷的“神器”——OpenClaw，号称能“一站式搞定多平台内容管理”。好奇心爆棚的我赶紧研究了一波，今天就来给大家扒一扒，这个听起来很厉害的 **OpenClaw** 到底是什么！

---

## 🔍 什么是 OpenClaw？

简单来说，OpenClaw 是一个**开源的多平台内容聚合与自动化工具**。你可以把它想象成一个“数字爪子”，能帮你从不同平台（比如社交媒体、博客、论坛）抓取、整理甚至同步内容，特别适合需要管理多个账号或做内容整合的小伙伴。

---

## ✨ OpenClaw 能做什么？

- **多平台内容抓取**：一键采集小红书、微博、公众号等平台的图文/视频。
- **内容自动整理**：根据标签、关键词自动分类，生成结构化的素材库。
- **跨平台同步发布**：编辑一次内容，可同步发布到多个关联账号（需平台允许）。
- **数据看板整合**：聚合各平台的阅读、互动数据，生成可视化报告。
- **自定义工作流**：通过简单配置，实现定时抓取、自动转存等个性化操作。

# OpenClaw 引导向导：开启 AI 助手的便捷之门

> 引导向导是在 macOS、Linux 或 Windows（通过 WSL2；强烈推荐）上设置 OpenClaw 的 **推荐** 方式。它在一个引导流程中配置本地网关或远程网关连接，以及频道、技能和工作区默认设置。

## 🚀 主要入口与模式

**主要入口点：**

```bash
openclaw-cn onboard
```

**重新配置：**

```bash
openclaw-cn configure
```

**快速开始 vs 高级模式**
向导提供两种启动模式：

* **快速开始**：采用默认设置，简化配置流程。
* **高级模式**：给予用户完全的控制权，展示每一个配置步骤。

## 🔧 向导的作用

### 本地模式（默认）

引导用户完成以下核心配置：

1. **模型/认证**：配置AI供应商的API密钥或认证方式。
2. **工作区**：设置工作区位置并生成启动文件。
3. **网关**：配置端口、绑定、认证及Tailscale暴露。
4. **频道**：连接如 WhatsApp、Telegram、Discord 等消息平台。
5. **守护进程**：安装系统服务（macOS的LaunchAgent或Linux的systemd用户单元）。
6. **技能**：启用和配置推荐技能。

### 远程模式

仅配置本地客户端以连接到位于其他地方的远程网关，不会在远程主机上进行安装或修改。

## 📝 核心流程详解（本地模式）

### 1. 现有配置检测

启动时，向导会检查 `~/.openclaw/openclaw.json` 是否存在，并允许用户选择 **保留**、**修改** 或 **重置** 现有配置。重置操作使用 `trash` 命令而非 `rm`，并提供不同范围的选择。

### 2. 模型与认证

向导支持多种AI供应商的认证配置：

* **Anthropic API 密钥**（推荐）
* OpenAI API 密钥
* MiniMax、Moonshot、火山引擎（豆包）等国内模型
* Vercel AI 网关、Cloudflare AI 网关等多模型代理方案
  向导会运行模型检查，并在配置的模型未知或缺少认证时发出警告。

### 3. 工作区

默认工作区路径为 `~/.openclaw/workspace`（可配置）。向导会为代理启动仪式生成所需的工作区文件。

### 4. 网关设置

配置网关的端口、绑定地址和认证模式。**建议即使对于本地回环连接也保持令牌认证**，以增强安全性。

### 5. 频道连接

支持连接多种主流即时通讯平台，如 WhatsApp、Telegram、Discord 等。对于私信安全，默认采用**配对（Pairing）**机制，首次私聊需要发送验证码进行授权。

### 6. 守护进程安装

* **macOS**：安装为 LaunchAgent（需要已登录的用户会话）。
* **Linux / Windows (WSL2)**：安装为 systemd 用户单元，并尝试启用 `linger` 以便用户在注销后网关服务仍能保持运行。

### 7. 健康检查

向导最后会启动网关（如果需要）并运行健康检查，确保服务配置正确、运行正常。

## 💡 实用提示

* 要添加更多独立的智能体（拥有独立的工作区、会话和认证），请使用：
  ```bash
  openclaw-cn agents add <name>
  ```
* 对于脚本或自动化场景，请使用 `--non-interactive` 参数。
* 推荐设置 Brave Search API 密钥，以便智能体可以使用 `web_search` 工具。最简单的方法是运行：
  ```bash
  openclaw-cn configure --section web
  ```
